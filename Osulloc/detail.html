OVERVIEW

다다일상, 오설록 과 함께하다

오설록은 제주의 자연을 이해하고 한국의 차 문화를 잇기 위해 시작되었습니다.
현재는 세계적으로 권위있는 명차 대회에서 꾸준히 수상을 이어가고 있습니다. 또한 차를 마심으로써 쉼을 갖고,
나를 이끌어내 차와 함께 삶의 아름다운 여백을 만들어 나가길 바라는 브랜드 입니다.

PROJECT GOAL

오설록 웹사이트 디자인

브랜드 스타일 유지

오설록 브랜드가 가지고 있는 차분한 느낌을 기반으로 제주의 자연과 숲의 감성을 담아낸 웹사이트 구현을 목표로 합니다.

반응형 웹사이트 개발

다양한 디바이스(PC, 모바일) 환경에서 최적화된 레이아웃과 사용성을 보장하는 반응형 웹사이트를 설계를 목표로 합니다.

학습 내용 활용

지금까지 학습한 내용을 활용하여 스크롤 애니메이션, 슬라이더 등을 통해 디자인에 생동감을 주고 구조적으로 간결하고 효율적인 코드를 작성을 목표로 합니다.

일상의 휴식을 전하는 오설록

일상의 휴식을 느낄 수 있는 차분한 디자인을 목표로, 절제된 표현과 제주의 ‘자연’, 특히 숲의 감성을 담아낸 디자인 키워드를 선정하였습니다.

PROBLEM & SOLUTION

01
모바일 메뉴 오픈 시 왜 헤더가 올라갈까?

모바일 화면에서 메뉴바가 전체 화면으로 열렸을 때 헤더의 스크롤 이벤트가 감지되어 애니메이션이 작동하는 오류 발생

hasClass와 return 함수를 활용하여 메뉴가 열릴때 (toggle-btn이 활성화 시) 스크롤 이벤트는 비활성화 시켜 애니메이션 동작을 차단

02
가로로 흐르는 텍스트 애니메이션은 어떻게 해야되는거지?

흐르는 텍스트 애니메이션을 구현하려고 하던 중, marquee 태그를 발견했으나 HTML5에서 지원 종료로 인해 사용이 불가능한 문제 문제 발생

css 애니메이션을 통해 흐르는 텍스트 효과 구현하여 marquee 태그 대체함

03
이미지 호버 시 텍스트를 표시 할려면 어떻게 해야할까?

HTML 구조상 존재하지 않던 텍스트를 이미지 호버 시 중간에 추가해야 하는 상황 발생. 모바일 화면에서는 텍스트가 표시되지 않고, PC 버전 이미지에 호버 시만 텍스트가 나타나도록 구현 필요

css의 after: content 속성을 사용해 HTML 구조를 수정하지 않고, 이미지 호버 시 동적으로 텍스트를 추가하여 문제 해결

RESULT

프로젝트 리뷰

  • 디자인 스타일의 변화

    지금까지는 아기자기한 요소가 많이 들어간 디자인을 주로 작업해왔으나, 이번 작업에서는 심플하고 간결한 웹사이트를 처음으로 제작해보았습니다. 디자인에서 요소를 많이 빼고 심플한 느낌을 유지하는 것이 예상보다 어려웠다는 점을 깨달았습니다. 하지만 결과적으로 깔끔하고 세련된 느낌을 잘 구현할 수 있었습니다.

  • 반응형 웹 디자인의 중요성

    반응형 웹페이지를 제작할 때 모바일과 PC 버전을 모두 고려하여 코드와 디자인을 조정하는 것이 중요하다는 점을 배웠습니다. 이를 위해 초기 설계 단계에서부터 충분히 고민하고, CSS 미디어 쿼리를 적절히 활용하는 방법을 배웠습니다.

  • 경로 설정의 중요성

    경로 설정의 중요성을 깨닫게 되었습니다. 특히, 외부 파일을 불러오는 경로를 잘못 설정하면 예상치 못한 오류가 발생할 수 있다는 점을 경험을 통해 배웠숩나더. 향후 작업 시에는 경로 설정을 두 번 이상 확인하여 실수를 방지해야겠다는 교훈을 얻게되었습니다.
